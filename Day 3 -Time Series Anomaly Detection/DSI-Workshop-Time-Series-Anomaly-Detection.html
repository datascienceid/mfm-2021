In this hands-on session, we will discuss 3 ways to detect anomaly in time series data :

  1. Simple Moving Average

  2. Standard Deviation + Moving Average

  3. Analyze the Residuals from Time Series Decomposition using anomalize package

1. Simple Moving Average

First, we will create a new column which is a % of difference between actual value and MA30 value as below.

df <- df %>% 
  mutate(pct_diff_actual_ma30 = (total_traffic - ma_30)/ma_30)

Then if we plot the distribution, we can see that the % of difference has normal distribution as below.

# Plot the Distribution of % Difference of Actual Value & MA30
d <- ggplot(df, aes(x = pct_diff_actual_ma30)) +
  geom_density() +
  theme(plot.title = element_text(hjust = 0.5)) +
  labs(title = '% Diff Actual & MA30 Distribution', x = '% Difference between Actual Value & MA30', y = '% Distribution')
  
plot(d)
## Warning: Removed 30 rows containing non-finite values (stat_density).

You are free to choose your own threshold, for example if for you the absolute difference “>25%” can already be considered as anomaly because it is suspicious, then goes with it.

For this example, we will use the Normal Distribution Anomaly Rules, that the anomaly are the data that falls outside 3 standard deviation.

Let’s calculate the upper and lower threshold of the % differences using Normal Distribution Anomaly Rules as stated before.

standard_dev <- sd(df$pct_diff_actual_ma30, na.rm = TRUE) # Calculate the Standard Deviation from the Distribution
upper_thre <- mean(df$pct_diff_actual_ma30, na.rm = TRUE) + 3*standard_dev # Upper Threshold, anything above 3 Standard Deviation
lower_thre <- mean(df$pct_diff_actual_ma30, na.rm = TRUE) - 3*standard_dev # Lower Threshold, anything below 3 Standard Deviation

print(upper_thre)
## [1] 0.3396436
print(lower_thre)
## [1] -0.3359468

We can see that the upper threshold is ~34% while the lower threshold is ~33.5%, that means if the % of differences is higher than ~34% or lower than ~33.5% then they will be classified as anomaly.

Now let’s add the anomaly_flag to the table.

## Adding new column which is a flag whether an observation classified as anomaly or not
anomaly_upper <- df$pct_diff_actual_ma30 > upper_thre
anomaly_lower <- df$pct_diff_actual_ma30 < lower_thre
anomaly_flag_raw = anomaly_upper | anomaly_lower

df <- df %>% 
  mutate(anomaly_flag_1 = anomaly_flag_raw)
df_anomaly_1 <- df %>% 
  filter(df$anomaly_flag_1 == TRUE) %>% 
  select(date_key, total_traffic, ma_30, pct_diff_actual_ma30) %>% 
  mutate(event = 'Anomaly')
print(df_anomaly_1)

2. Standard Deviation + Moving Average

Another approach instead of using %of Difference is using Standard Deviation.

This approach is very similar to the 1st approach, but instead of presenting the difference in terms of % from MA30, we will measure the differenece in terms of standard deviation unit.

By using % of Difference, the threshold is always same over the time. So imagine if at one time it is normal that our traffic is being volatile, that means many data point will captured as anomaly. But using standard deviation means we also adjust the threshold over the time

Let’s first add a new column that is the standard deviation of Last 30 Days, and then add another column which is the difference of actual value and MA30 in standard deviation unit.

df$sd_30 <- rollapplyr(df[,2],list(-(30:1)), sd, fill=NA) #Calculating Standard Deviation for Last 30 Days

df <- df %>% 
  mutate(sd_diff = (total_traffic - ma_30)/sd_30)

3. Using anomalize Package

Let’s first create a new dataframe for this approach,as this package needs the input in form of tibble.

df_new = read.csv('C:/Users/nakama/Documents/time_series_sample_data_2.csv')
df_new$date_key <- as.Date(df_new$date_key,format="%Y-%m-%d")
df_new <- as_tibble(df_new) #anomalize package need input to be in tibble format

On time series decomposition, there are several time series components :

  1. Seasonal

  2. Trend

  3. Remainder

\[y_t = S_t + T_t + R_t \]

\[y_t : Actual Value\] \[S_t : Seasonal\] \[T_t : Trend\] \[R_t : Remainder\]

The decompisition here is using STL method. STL stands for Seasonal and Trend decomposition using Loess as already explained in session before.

anomalize have 2 options method in detecting residuals that are anomalies :

  1. IQR : InterQuartile Range, it is a similar method used in tsoutliers() function of the forecast package. In IQR a distribution is taken and 25% and 75% inner quartile range to establish the distribution of the remainder. Limits are set by default to a factor of 3 times above, and below the inner quartile range, any remainder beyond the limit is considered as an anomaly.

  2. GESD : Generalized Extreme Studentized Deviate Test, in GESD anomalies are progressively evaluated removing the worst offenders and recalculating the test statistics and critical values, or more simply you can say that a range is recalculated after identifying the anomalies in an iterative way.

Both IQR and GESD have their pros and cons, IQR is faster as there are no loops involved but it’s not as accurate as GESD since the anomalies skews the median which is removed in GESD.
